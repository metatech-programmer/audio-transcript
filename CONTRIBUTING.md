# Contributing

Thanks for your interest in contributing. Please follow these guidelines:

- Code style: run `npm run format` before committing. We've added Prettier configuration to keep formatting consistent.
- Linting: run `npm run lint` and `npm run type-check` where applicable.
- Commit messages: use clear, concise messages. Follow conventional commits if possible.
- Branching: create feature branches named `feat/your-short-description` or `fix/short-description`.
- Pull requests: include a short description of the change, testing steps, and link any related issue.

CI runs on each PR and will run build + lint checks. Thanks!
